# Privacy & Security

## 原则

1. 默认本地处理与本地存储。
2. 只访问用户明确选择、合法拥有的数据。
3. 发送给模型的上下文最小化。
4. 原始事实、模型生成内容和秘密分层存储。
5. 公开发布前使用全新历史和合成数据，而不是“删掉当前文件就算脱敏”。

## 公开仓库不包含

- 真实聊天正文、联系人标识或数据库；
- 真实 AI 对话和关系报告；
- API Key 或可还原 Key 的密文；
- 私密 EXE 与构建种子；
- 原生聊天密钥恢复实现；
- 用户电脑绝对路径和运行日志。

## 本地服务保护

- loopback-only；
- 同源会话 Cookie；
- Host / Origin 校验；
- 请求大小与读取超时限制；
- SQL 工具只允许只读查询；
- 端口独占与构建身份检查。

## 模型接口

公开版支持 OpenAI-compatible 接口。不要把 Key 写入仓库或 `.env.example`。生产 Windows 版把 Key 保存在后端，并使用当前 Windows 用户绑定的加密存储。

## 威胁模型的边界

本地优先不能防止已经控制当前操作系统账户的攻击者读取文件或进程内存；它主要减少无意上传、网页跨源访问、安装包误分发和不同关系串库风险。

## 负责任使用

本项目用于分析用户自己合法持有并获得适当同意的数据。不要用于绕过授权、监控他人或违反平台条款的行为。

发现安全问题时，请不要在 Issue 中粘贴真实聊天、Key 或绝对私人路径。可以只描述复现步骤和脱敏错误类型。
